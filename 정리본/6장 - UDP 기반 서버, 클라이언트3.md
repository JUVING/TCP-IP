
# *6-1*

UDP와 TCP 비교
1. UDP 소켓은 *SEQ, ACK와 같은 메시지 전달을 하지 않는다.*(Flow control 없다)
2. *연결과 해제 과정이 없다.
3. 데이터 *분실, 손실 위험*이 있다.
4. 확인 과정인 없어 *전송이 빠르다.
5. 안전성보다 *성능이 중요할때* 사용한다.

# *6-2*

TCP 는 소켓이 연결된 위치로 데이터를 전송하라고 명령 하지만
UDP는 소켓과 데이터를 *전송 해야할 주소 값*을 같이 전송한다.

UDP는 연결이란 개념이 없고 서버 소켓과 클라이언트 소켓의 구분도 없다.
